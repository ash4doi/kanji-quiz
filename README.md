# かんじ よみクイズ

小学生むけの漢字「読み」学習ウェブアプリ。Duolingo のような短いセッションで、漢字の読み（ひらがな）をくり返し練習する。書き取りはなく、読みだけに特化している。

公開URL: https://ash4doi.github.io/kanji-quiz/

## 特長

- **対象**: 小学1年生（80字）・2年生（160字）の漢字。音読みと訓読みは別問題として出題する。
- **やさしさ重視**: 基本的な語と例文だけを使い、難読・特殊な読みは扱わない。数の訓読みなどは送り仮名をヒントにしている（例「三つ」→ こたえ「みっ」）。
- **出題のしかた**: 中央に漢字を使った言葉を大きく表示（対象漢字をハイライト）、下に例文を2つ、その下にひらがな入力欄。回答すると ◯/✕・正しい読み・例文のふりがな（ルビ）を表示する。
- **セッション**: 1回 10問 / 20問 / 全問、出題順は「バラバラ / じゅんばん / にがて」から選べる。
- **にがて優先**: 記録をもとに、未出題・正解率の低い・しばらく解いていない問題を優先して出す。
- **記録**: 連続学習日数（ストリーク）・出題回数・正解率をブラウザに保存。学年ごとに別管理。
- **バックアップ**: 記録を JSON で書き出し／読み込みできる。
- **サーバー不要**: 静的ファイルのみ。ログインなし。

## ファイル構成

- `index.html` — アプリ本体（HTML/CSS/JS が1ファイル）。起動時にCSVを読み込む。
- `kanji_g1.csv` — 1年生の問題データ（115問）。
- `kanji_g2.csv` — 2年生の問題データ（178問）。
- `design_notes.md` — 設計メモ。
- `kanji_list_s.txt` — 学年別の配当漢字リスト（元データ）。

公開（GitHub Pages）に必要なのは `index.html` と 2つの CSV の計3ファイル。

## データ形式（CSV）

1問が1行。1行目はヘッダ。列は次の通り。

| 列 | 意味 |
|----|------|
| `id` | 問題の識別子。記録のキーになるので変更しない（例 `火-音`） |
| `kanji` | 対象の漢字1字 |
| `type` | `音` または `訓` |
| `yomi` | 正解の読み（ひらがな。完全一致で判定） |
| `word` | 中央に大きく表示する言葉（送り仮名を含めてよい） |
| `highlight` | `word` の中で読みを問う漢字 |
| `ex1` / `ex1_yomi` | 例文1と、その中の対象漢字の読み |
| `ex2` / `ex2_yomi` | 例文2と、その中の対象漢字の読み |

文字コードは BOM 付き UTF-8（Excel / Google スプレッドシートで開いても文字化けしない）。

## 問題の更新・追加

ビルド工程はない。CSV を直接編集する。

1. `kanji_g1.csv` または `kanji_g2.csv` を Excel・スプレッドシート・テキストエディタで編集する。
   - 修正は該当行を直す。追加は同じ形式で行を足す。`id` は学年内で重複させない。
2. 編集した CSV を GitHub の `kanji-quiz` リポジトリにアップロードして上書きする。
3. アプリを再読み込みすると反映される（読み込み時にキャッシュを回避しているため、すぐ反映される）。

`index.html` を触る必要はない。

### 学年を増やすには

1. 同じ列構成で `kanji_g3.csv` などを作る。
2. `index.html` 内の `GRADE_CSV` に1行足す（例 `"3": "kanji_g3.csv"`）。
3. 開始画面の学年ボタン（`gradeOpts`）に `<button class="opt" data-grade="3">3ねん</button>` を足す。

## ローカルでの確認

`index.html` は CSV を `fetch` で読むため、ファイルを直接開く（`file://`）と動かない。簡易サーバー経由で開く。

```
cd <このフォルダ>
python3 -m http.server 8000
```

ブラウザで `http://localhost:8000/` を開く。

## iPad での使い方

公開URL（https）を Safari で開き、共有メニューから「ホーム画面に追加」すると、アイコンからアプリのように起動できる。

## 記録について

- 記録はブラウザの localStorage に保存される。**端末・ブラウザごと**に別管理で、サーバーには保存されない。
- 学年ごとに別キー（`kanji_v2_g1` / `kanji_v2_g2`）で保存する。
- 別の端末やブラウザとは共有されない。引き継ぎたいときは「きろくを書き出す／よみこむ」を使う。
- Safari は、サイトを約7日間開かないと localStorage を消すことがある。長期保存はエクスポートを併用すると安全。

## 技術メモ

- プレーンな HTML / CSS / JavaScript のみ。ビルドツール・依存ライブラリ・バックエンドなし。
- 判定はひらがなの完全一致。
- ふりがな表示は `<ruby>` タグを使用。
